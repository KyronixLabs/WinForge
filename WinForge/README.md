# WinForge

WinForge is the ultimate all in one Windows PC optimisation tool.

It is a Windows desktop app for improving, maintaining, repairing, cleaning, and monitoring Windows 10 and Windows 11 PCs from one clean interface.

WinForge is designed for users who want more control over their PC without digging through hidden Windows menus, old control panels, PowerShell commands, registry paths, system tools, and scattered settings pages.

WinForge brings optimisation, cleanup, startup control, gaming tuning, network tools, privacy checks, security scans, repair actions, restore tools, and system reports into one app.

## Repository description

The ultimate all in one Windows PC optimisation tool for Windows 10 and Windows 11. WinForge helps clean, tune, repair, secure, monitor, and optimise a Windows PC through a clean desktop interface.

## No fake optimisation

WinForge is designed to be practical, transparent, and reversible where possible.

Many PC optimiser apps use fake or risky tricks that sound impressive but often do very little, make Windows less stable, or reduce security.

WinForge avoids those behaviours.

## Fake optimisation methods WinForge avoids

## Fake registry cleaning

Some optimiser apps claim that deleting thousands of registry entries will make a PC much faster.

In reality, most unused registry entries have no noticeable impact on performance. Aggressive registry cleaning can also break apps, file associations, drivers, services, and Windows features.

WinForge does not perform mass registry cleaning.

When WinForge changes a registry backed setting, it is a specific known setting with a clear purpose.

## RAM cleaning

Some apps advertise a memory cleaner or RAM booster.

These tools often force Windows to empty useful cached memory. Windows already manages memory intelligently, and clearing cache can actually make apps reopen slower because Windows has to reload data again.

WinForge does not pretend that empty RAM automatically means better performance.

## Disabling security for speed

Some tools disable Windows Defender, firewall protection, SmartScreen, or Windows Update to claim a performance boost.

That is not proper optimisation. It makes the PC less safe.

WinForge does not disable important Windows security features for fake speed gains.

## Disabling Windows Update permanently

Some optimiser tools turn off Windows Update completely.

This can leave the system missing security fixes, driver fixes, and stability updates.

WinForge does not permanently disable Windows Update.

WinForge focuses on update cleanup, restart awareness, active hours, and repair tools instead.

## Random service disabling

Some apps disable large lists of Windows services without checking what the PC actually uses.

This can break printing, Bluetooth, networking, gaming services, Store apps, updates, file sharing, development tools, or accessibility features.

WinForge avoids blind service disabling.

Service tools are treated as review tools, not random one click damage tools.

## FPS miracle tweaks

Some apps promise huge FPS increases through mystery registry changes, timer tweaks, or hidden gaming hacks.

Most of these changes are either placebo, outdated, hardware specific, or risky.

WinForge focuses on real gaming improvements such as Game Mode, power settings, background capture, network power saving, mouse acceleration, shader cache cleanup, and selected game GPU preference.

## Driver installer shortcuts

Some optimiser apps push unknown driver installers or third party driver packs.

Bad drivers can cause crashes, blue screens, broken audio, broken networking, or poor gaming performance.

WinForge does not install random drivers.

WinForge can report device problems and help export drivers, but driver updates should come from Windows Update, the PC manufacturer, or the hardware vendor.

## Deleting user files without review

Some cleaners remove files too aggressively.

WinForge is designed to focus on safe cleanup areas and reports. Storage tools should show what is being reviewed so important personal files are not removed blindly.

## Killing random processes

Some boosters kill background processes without understanding what they are.

This can close unsaved work, break sync tools, stop game launchers, interrupt updates, or damage running tasks.

WinForge avoids killing unknown system processes. Gaming tools can close selected background apps only when the user chooses that behaviour.

## Mystery tweaks

WinForge is built around clear actions.

Each optimisation should explain:

1. What it changes
2. Why it may help
3. Whether administrator access is needed
4. Whether a restart may be required
5. Whether it can be reverted
6. Whether it is safe, careful, standard, or advanced

The Select safe profile button chooses a safer set of recommended options.

## Tutorial

## Step 1

Open WinForge.

For full functionality, run it as administrator.

Administrator access is needed for many system level actions such as power settings, firewall changes, repair tools, service checks, restore point creation, and all users startup entries.

## Step 2

Open the Dashboard.

Review the basic system information.

Check:

1. Administrator status
2. Windows version
3. System drive space
4. Memory information
5. Local network information
6. Uptime

## Step 3

Open Optimise.

Select a profile or manually choose optimisation toggles.

For a first run, use the safe profile.

Review the selected options before applying.

Press Optimise to apply the selected changes.

## Step 4

Review the result output.

If WinForge reports that a restart is needed, restart the PC after saving work.

## Step 5

Open Apps and Startup.

Search through startup entries.

Disable apps that do not need to start with Windows.

Press Apply startup changes.

## Step 6

Open Storage.

Run storage reports before deleting anything.

Use cleanup tools for temporary files and caches.

Review Downloads and large file reports for personal cleanup decisions.

## Step 7

Open Gaming if the PC is used for games.

Use Apply Gaming Optimisation for a gaming focused setup.

Use Choose game exe and Set high performance GPU preference for a selected game where supported.

Use Start Gaming Session before playing and End Gaming Session afterwards if temporary session behaviour is wanted.

## Step 8

Open Network if there are connection issues.

Use DNS flush, network checks, proxy review, VPN checks, hosts file review, open ports, and firewall review tools as needed.

## Step 9

Open Security.

Run security scans and review Defender, firewall, accounts, baselines, permissions, persistence locations, and remote access risks.

## Step 10

Open Repair and Recovery if Windows is behaving incorrectly.

Use SFC, DISM, Windows Update repair, restore point, driver export, and recovery tools carefully.

Some actions can take several minutes.

## Step 11

Open Restore Centre if a reversible optimisation needs to be undone.

Choose Revert latest or select a saved snapshot.

## Step 12

Open Logs.

Review action output if something did not work as expected.

Logs are useful for troubleshooting and support.

## Building the app

WinForge can be built with Visual Studio or with the included build script.

## Build with Visual Studio

1. Open `WinForge.sln`
2. Select the WinForge app project
3. Build the solution
4. Run the app

## Build with the script

Run this file from the repository root:

```text
Build-WinForge.bat
```

The build script publishes the app into the publish folder.

## Requirements

WinForge is built for Windows.

Recommended environment:

1. Windows 10 or Windows 11
2. Visual Studio 2022
3. .NET desktop development workload
4. .NET 8 or newer
5. Administrator access for full optimisation features

## Folder layout

WinForge stores data in the Windows ProgramData area by default.

The default data path is:

```text
C:\ProgramData\WinForge
```

This folder can contain:

1. Logs
2. Reports
3. Snapshots
4. Profiles
5. Startup state
6. Exported reports
7. Support bundles
8. Maintenance scripts

## Snapshots and revert

WinForge saves information about optimisation runs where possible.

A snapshot can include:

1. Time of the run
2. Selected options
3. Applied actions
4. Revert scripts where available
5. Output logs
6. Restart notes

The Revert feature uses available revert actions from the saved snapshot.

Not every Windows action can be perfectly reverted, so WinForge labels actions carefully.
